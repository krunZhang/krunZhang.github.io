    比如下面这个例子:

    class Point { int[] metrics; }
interface Move { void move(int deltax, int deltay); }

    此时声明了一个 Point 类类型，一个 Move 接口类型，还有 Point 中的属性 metrics 是一个 int[] 整数数组类型。

    一个类或接口由一个标识符或点和标识符组成，其中每个标识符都可以跟一个类型参数。如果类型参数出现在类或接口的任何地方，那么这个类或接口是一个参数化类/参数化接口，即泛型。

    类或接口中的每个标识符都被分割为包名或类型名。被分类为类型名的标识符可能被附带一个注解。如果一个类或接口的形式为 T.id (可选的跟在类型参数的后面)，那么 id 必须是 T 的可访问成员类型的简单名称，否则编译时会抛出错误。类或接口表示成员类型。

    对象

    一个对象可以是一个类的实例或是一个数组。

    引用值 (reference values) （通常只会是一个引用）是指向这些对象的指针，或是一个特殊的空引用。空引用表示没要引用到任何对象。

    类的实例由类的实例创建表达式显示创建。

    数组通过数组的创建表达式显示创建。

    在非常量表达式中使用字符串连接运算符 + 时，会隐式得创建新的类实例，从而生成 String 类型的新对象。

    当一个类或接口被初始化，创建一个类的新实例，执行一个局部变量的声明语句时，都可能会隐式得创建新的数组对象，因为其初始化表达式被执行了。

    Boolean、Byte、Short、Character、Integer、Long、Float、Double 这些类型的新对象则会在装箱拆箱时被隐式得创建。

    比如下面这个例子：

    class Point {
    int x, y;
    Point() { System.out.println("default"); }
    Point(int x, int y) { this.x = x; this.y = y; }

    / *一个Point实例在类初始化被创建* /
    static Point origin = new Point(0,0);

    / *由一个字符串拼接运算符隐式得创建新对象* /
    public String toString() { return "(" + x + "," + y + ")"; }
}

class Test {
    public static void main(String[] args) {
        // *通过 newInstance 显式得创建对象实例* /
        Point p = null;
        try {
            p = (Point)Class.forName("Point").newInstance();
        } catch (Exception e) {
            System.out.println(e);
        }

        / * 在数组的初始化语句中隐式得创建了新对象实例* /
        Point a[] = { new Point(0,0), new Point(1,1) };

        / *由一个字符串拼接运算符隐式得创建新对象* /
        System.out.println("p: " + p);
        System.out.println("a: { " + a[0] + ", " + a[1] + " }");

        / *由一个数组创建表达式创建数组实例* /
        sa[0] = "he"; sa[1] = "llo";
        System.out.println(sa[0] + sa[1]);
    }
}

    上述代码将输出:

    default
p: (0,0)
a: { (0,0), (1,1) }
hello

    对象的操作符

    • 属性访问，使用限定名或属性访问表达式
    • 方法调用
    • 类型转换操作符
    • 字符串拼接运算符 +， 当给定 String 和一个引用对象时，会调用引用对象的 toString 方法来得到一个字符串（引用为空或 toString 返回空时则会得到字符串 null），然后创建一个值为两个字符串相接的新的字符串实例
    • instanceof 运算符
    • 引用相等运算符 == 和 !=
    • 条件运算符 ? :

    多数情况下，会有多个变量持有的都是到同一个对象的引用。大多数对象都持有状态，这些状态被存储在类实例的对象的字段或是数组对象的单元变量中。如果两个变量包含对同一对象的引用，那么其中一个变量对引用对象的状态进行修改后，另一个对象都可以观察到修改后的状态。

    考虑下面的代码：

    class Value { int val; }

class Test {
    public static void main(String[] args) {
        int i1 = 3;
        int i2 = i1;
        i2 = 4;
        System.out.print("i1==" + i1);
        System.out.println(" but i2==" + i2);
        Value v1 = new Value();
        v1.val = 5;
        Value v2 = v1;
        v2.val = 6;
        System.out.print("v1.val==" + v1.val);
        System.out.println(" and v2.val==" + v2.val);
    }
}

    上述代码将输出:

    i1==3 but i2==4
v1.val==6 and v2.val==6

    因为 v1.val 和 v2.val 引用的是由 new 创建的同一个 Value 类的实例对象，而 i1、i2 则是不同的变量。

    每个对象都会与一个监视器关联，因为 synchronized 方法和 synchronized 语句需要监视器来提供对多个线程下并发访问对象状态的控制。

    Object 类

    Object 是其他所有类的父类。

    所有的类和数组类都继承自 Object，因此它们都含有 Object 的所有方法:

    • clone 方法用于复制对象

    • equals 方法定义了如何比对两个对象是否相等，它比较的是值而不是引用

    • finalize 方法会在对象被销毁之前执行

    • getClass 方法会返回一个 Class 对象以表示对象的类型

      所有的对象都可以获取其 Class 对象，它可以用来查看类的信息，比如类的完整路径名称、属性成员、它的直接父类和所有它实现的接口。

      getClass 方法返回的是 Class<? extends |T|>，其中 T 是调用此方法的类或接口，|T| 表示泛型擦除。

      如果某个类的方法被用 synchronized 所修饰，那么监视器会将这个方法与这个类的 Class 对象绑定。

    • hashCode 是一个非常有用的方法，它常常与 equals 方法配合使用，java.util.HashMap 之类的哈希表类也依靠它进行工作。

    • wait、notify 和 notifyAll 被用来在多线程下并发编程

    • toString 方法会返回此对象的字符串形式

    String 类

    String 的实例是 Unicode 编码序列 的表现形式。

    String 实例是不可变的，这体现在对其值进行的任何修改都会产生一个新的 String 对象实例。

    所有的字符串量都是 String 对象实例的隐式引用。

    当字符串拼接运算符 + 的结果不会是一个常量时，会创建一个新的 String 对象来容纳新值。

    当类型相同的时候

    如果两个类拥有相同的二进制名字和它们可能持有的所有类型参数也都是一样的时候，则编译时视为它们是同一种类型。

    当两种引用类型相同是，会被认为是同一个类或同一个接口。

    在运行期，相同的两个引用类型是可能被不同的类加载器加载的，此时则认为它们不是同一个类型。

    以下情况将把两种引用类型视为相同：

    • 它们都是类类型或都是接口类型，且被同一个类加载器所定义，同时还拥有相同的二进制名称，这样就会称它们为相同的 运行时类 (run-time class) 或 运行时接口 (run-time interface)
    • 它们都是数组类型，且它们的单元类型是相同的运行时类型
